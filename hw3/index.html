Homework 3 Milestone B

 

Description

Wordle with a musical twist!

Milestone A

Research

Many digital sequencers (such as those that come by default in DAWs) are pretty much straightforward grids, most likely because they need to be practical and accessible to the majority of users. This ease-of-use is good design, but it isn't exactly interesting from an artistic perspective.

Searching for more creative uses of sequencers, I looked at Eurorack modules, where there is usually some traditional concept being used in a unique way. I own the book Patch & Tweak - Exploring Modular Synthesis and reread the section on sequencers to get some new ideas. Here are just a few:

  • Make Noise's René: a "3D" sequencer where sequences are stored in cartesian coordinates and X, Y, and Z dimensions can be stepped through independently in different orders. Time does not have to move "left to right" (but can of course), but can step through the 3D cube based on different patterns and algorithms.
  • Pamela's Pro Workout: This is more of a clock generator than a sequencer, but it can be used to trigger steps in a sequence in unique timings. I particularly like the Euclidean patterns generator which takes a sequence length (e.g. 20), the number of "hits" (e.g. 11) and "produces a pattern based on the hits being as equidistant from each other as possible." These patterns can be rotated (i.e. changing the start position), padded with rests, and have the hits have a probability of not triggering.
  • Kilpatrick K4815 Pattern Generator: This sequencer generates notes and rhythms by using a musical scale and a visual pattern. The user only controls the "big picture" (how the sequencer reacts), not the actual notes themselves. Instead, an algorithm determines which notes are placed where based on these general parameters.

Ideas

1. Chordle (Chuck + Wordle) 

  • Each game of wordle represents a sequence (5 letter word + 6 chances to guess would be max 30-step sequence; 3 letter word + 3 chances to guess would be max 9-step sequence).
  • Multiple games of varying lengths can be played at once, allowing for both polyphony and polyrhythms. (You could move between the different games, you don't have to finish one game to move on to the next).
  • Green (i.e. correct letter in the correct spot) means always trigger, Yellow (i.e. correct letter in an incorrect spot) triggers with a certain probability, Gray (i.e. incorrect letter) is a rest (no trigger)
  • Games are stepped through based on different patterns (it doesn't have to be top-left to bottom-right). 
  • Typing speed determines the tempo
  • Guesses (and specifically the letters of a guess) determine a bunch of different things (this is where I haven't figured out everything yet). Some possibilities:
    • A-Z could map to notes (such as % 12 for a little over 2 octaves of 12 notes) OR Letters could map to different samples
    • Different letters could be modifiers (I - increase, D - decrease for volume/pitch/whatever; S - sharp, etc.) that would only activate when there was a trigger (i.e. letter in the correct spot)
    • The frequency of letters used in a game (such as 25% As, 10% Bs, etc.) could correspond to how often those notes are played
  • Game progress could also modify the sound (how many correct guesses, how many green vs. yellow squares, etc.) which could determine things like yellow square hit-probability, the sequence pattern to follow, and more.
  • Possible modifiers
    • Shift individual rows / columns.
    • Transpose matrix?
    • Rotate words?
  • A crazy idea that Kunwoo and I discussed:
    • 3D wordle? Take the Rubik's cube idea (below) and mix it with wordle
    • Each "cube" is associated with only one word, which would give you six ways to get to that word (i.e. six planes of differing sequence patterns). In this way, if you "beat" one side and learn the word, it isn't really about winning when playing the other sides for the same cube, but instead, it's about getting to that word in an interesting way (creating a unique sequence).
    • Can possibly rotate columns and rows (like a Rubik's cube) to further create differing sequence values.

2. Roads

  • Create multiple roads in parallel, where each road length corresponds to sequence length
  • Cars drive on the road, triggering events
  • The speed at which the cars drive is the tempo, the position of cars (how far apart they are from each other) could map to swing
  • Multiple cars (playheads) could drive on a single road (sequence), triggering elements in a single sequence in unique rhythms
  • Roads can have inclines (corresponding to attack), plateaus (sustain), and declines (decay). The height of these roads could correspond to amplitude (and create a typical ASR envelope) or even map to pitch (where inclines / declines map to a glide/slew for moving between pitches)
  • Roads could have different "weather conditions", icy roads, rain, potholes, etc. that could affect timbral parameters, such as adding distortion, reverb, delay, etc.

3. Rubik's cube

  • Similar to my 3D wordle idea but I actually thought of this one first.
  • Each color corresponds to a sample (or rest). The sequencer steps through the current face of the rubik's cube.
  • You can have multiple Rubik's cubes at a time of different sizes: 2x2, 3x3, 8x8, etc.
    • Use even weirder Rubik's cube shapes? They come in hexagons, pyramids, ovals, etc. which could change the step pattern.
  • The user can manipulate each of the available Rubik's cubes by swapping to a different face, rotating the current face (counter-)clockwise, or rotating any of the horizontal or vertical "slices". Any of these actions will change the sequence.
